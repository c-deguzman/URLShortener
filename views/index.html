API Reference: URL Shortening Microservice

URL Shortening Microservice- used for shortening urls and calling on shortened urls.
Two classes of call:

  • a shortening request
  • a shortened retrieval

Shortening Request

A shortening request is based upon up to three parameters, which may be given in any order

https://misty-iris.glitch.me/serv?base=mybase&desired-short=shorty&override=true
  • base - url sent in that will be shortened. Unless override is active, must be in proper http or https format
  • desired-short - (optional) (case-sensitive) if available, shortened key will be desired-short, error if otherwise. MUST contain at least one alphabetical character
  • override - (optional) if set to 'true', will ignore url format checking. Note: your shortened url may not redirect properly if base is not a proper url
Example Call Example Result
https://misty-iris.glitch.me/serv?desired-short=red&base=https://www.reddit.com {"short":"https://misty-iris.glitch.me/short/red","base":"https://www.reddit.com"}
https://misty-iris.glitch.me/serv?base=https://www.reddit.com {"short":"https://misty-iris.glitch.me/short/15","base":"https://www.reddit.com"}
https://misty-iris.glitch.me/serv?base=httpqqr35s://www.reddit.com {"error":"base url not formatted correctly"}
https://misty-iris.glitch.me/serv?base=httpqqr35s://www.reddit.com&override=true {"short":"https://misty-iris.glitch.me/short/10","base":"httpqqr35s:/www.reddit.com"}
https://misty-iris.glitch.me/serv?base=https://www.reddit.com&desired-short=red {"error":"name already taken"}

Shortening Retrieval

A retrieval request is based upon the provided key.

https://misty-iris.glitch.me/short/11

If key is found, page will redirect to proper site (if possible, see *override), otherwise will raise error
